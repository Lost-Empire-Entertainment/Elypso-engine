This doc highlights ideas for a framebuffer for opengl if i consider adding it to KalaGraphics

main:
- framebuffer object (fbo) as u32
- width and height as vec2
- msaa samples as u8
- color textures as u32 texture IDs
- color render buffers as u32 (only if MSAA is used)
- depth/stencil attachments
- initialize as:
	- color-only fbo
	- fbo with depth
	- msaa
	- multiple color attachments
	- recreate after resize (hot reload?)
- bind, unbind, bind screen (?)
- blit to target framebuffer (?)
- resolve to texture (gl texture ID)
- get color and depth texture

misc ideas:
- downsampling
- supersampling
- post processing effects
- screen space effects
